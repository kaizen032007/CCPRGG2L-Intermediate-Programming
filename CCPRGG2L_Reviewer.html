Introduction to Java ¬∑ Setting Up the Environment ¬∑ Java Syntax

What is Java?

Java is a high-level, class-based, object-oriented programming language designed to have as few implementation dependencies as possible.

  • Follows the principle of "Write Once, Run Anywhere" (WORA) ‚Äî compiled Java code runs on any platform that supports Java without recompilation.
  • Developed in 1991 by James Gosling, Mike Sheridan, and Patrick Naughton at Sun Microsystems.
  • Originally called "Oak" (after a tree outside Gosling's office); later renamed "Java" after Indonesian coffee.
  • Sun Microsystems was later acquired by Oracle Corporation.

Key Milestones of Java

1991

Project initiated by James Gosling and the Green Team. Originally designed for interactive television.

1995

Java officially launched as Java 1.0 with the WORA slogan.

1997

JDK 1.1 released ‚Äî added JDBC API, JavaBeans, and inner classes.

1999

Java 2 (J2SE 1.2) released ‚Äî introduced J2SE, J2EE, J2ME editions; Swing GUI and Collections framework.

2004

Java 5.0 ‚Äî generics, metadata, enums, enhanced for-loop.

2006

Java released under GNU GPL ‚Üí OpenJDK born.

2009

Oracle Corporation acquired Sun Microsystems (and Java).

2014

Java 8 ‚Äî lambda expressions, Stream API, new date/time API.

2017

Oracle announced a new 6-month release cadence.

Present

Java continues to evolve; widely used in web, mobile, and enterprise systems.

Real-World Applications of Java

  • Web Applications ‚Äî server-side apps using Spring, Hibernate, Struts.
  • Enterprise Applications ‚Äî large-scale systems using Java EE / Jakarta EE.
  • Mobile Applications ‚Äî most Android apps are written in Java.
  • Big Data ‚Äî Apache Hadoop, Apache Spark.
  • Scientific Applications ‚Äî data processing, simulations.
  • Gaming ‚Äî 2D/3D and mobile games.
  • Cloud Computing ‚Äî AWS, Google Cloud, Azure.

Setting Up the Java Development Environment

JDK (Java Development Kit) ‚Äî required to compile and run Java programs.

  • Download from Oracle's official site or OpenJDK.
  • After installing, set the PATH environment variable to the JDK's bin directory (e.g., C:\Program Files\Java\jdk-<version>\bin).
  • Verify with the command: javac -version

Popular IDEs:

  • Eclipse ‚Äî widely used, open-source.
  • IntelliJ IDEA ‚Äî known for advanced code editing and refactoring.
  • NetBeans ‚Äî easy to use, built-in support for Java SE/EE.

Control Flow Statements

Overview of Control Statements

Control statements allow the flow of execution to be controlled within a program. Java has three types:

  • Decision-Making Statements ‚Äî if, if-else, else-if ladder, switch
  • Looping (Iteration) Statements ‚Äî while, do-while, for, enhanced for
  • Branching Statements ‚Äî break, continue, return

Decision-Making: if / else-if / else

// Simple if if (age >= 18) { System.out.println("Adult"); } // else-if ladder if (marks >= 90) { System.out.println("Grade: A+"); } else if (marks >= 80) { System.out.println("Grade: A"); } else { System.out.println("Grade: B"); }
Key Rule The first condition that evaluates to true is executed; all others are skipped.

Decision-Making: switch Statement

A multi-way branch ‚Äî selects a block of code based on the value of an expression.

switch (day) { case 1: System.out.println("Monday"); break; case 2: System.out.println("Tuesday"); break; default: System.out.println("Invalid day"); }
Important Always include break after each case to prevent "fall-through" (executing the next case unintentionally).

Looping: while Loop

Executes a block of code as long as the condition is true.

int i = 1; while (i <= 5) { System.out.println("Count: " + i); i++; }

Looping: do-while Loop

Like while, but guarantees the block runs at least once ‚Äî condition is checked after execution.

int i = 1; do { System.out.println("Count: " + i); i++; } while (i <= 5);

Looping: for Loop

Used when you know exactly how many times to repeat.

for (int i = 1; i <= 5; i++) { System.out.println("Count: " + i); }

Enhanced for Loop (for-each) ‚Äî iterates over arrays or collections:

int[] numbers = {10, 20, 30, 40}; for (int number : numbers) { System.out.println(number); }

Branching Statements

Arrays

What is an Array?

An array is a data structure that stores multiple values of the same type in a single variable.

Size

Element Types
Same Type

  • Fixed size: Once declared, size cannot be changed.
  • Same type: All elements must be the same data type.
  • Indexed access: Elements accessed using an index, starting at 0.

1. One-Dimensional Array

A simple list of elements organized in a single row. Accessed using a single index.

// Declaration and initialization int[] scores = {85, 90, 78, 92, 88}; // Accessing elements System.out.println(scores[0]); // prints 85 System.out.println(scores[4]); // prints 88 // Iterating with for-each for (int s : scores) { System.out.println(s); }

Ideal for: linear data like scores, prices, or names.

2. Two-Dimensional Array

Like a grid or table ‚Äî rows and columns. Each element is identified by two indices: [row][column].

// Declaration int[][] matrix = new int[3][3]; // Access element at row 1, column 2 matrix[1][2] = 5; // Iterating with nested for loops for (int i = 0; i < 3; i++) { for (int j = 0; j < 3; j++) { System.out.print(matrix[i][j] + " "); } System.out.println(); }

Useful for: grids, matrices, board games (e.g., chess), multiplication tables.

Introduction to Classes, Methods & Exception Handling

Classes and Objects

  • Class ‚Äî a blueprint or template that defines the structure and behavior of objects. Contains attributes (fields) and methods.
  • Object ‚Äî an actual instance of a class. Created using the new keyword.
  • Method ‚Äî a block of code within a class that performs a specific task (like a function in other languages).
Analogy A "Car" class is a blueprint. A specific car like "Honda Civic" is an object (instance) of that class.
// Defining a class public class Car { String color; // attribute String model; void drive() { // method System.out.println("Driving..."); } } // Creating an object Car myCar = new Car(); myCar.color = "Red"; myCar.drive();

Types of Methods

TypeDescriptionKey Keyword
Instance MethodBelongs to an object; called using object reference(none)
Static MethodBelongs to the class; called without creating an objectstatic

Method Structure:

  • Modifiers: public, private, static, etc.
  • Return Type: void, int, String, etc.
  • Method Name: Follows camelCase naming convention.
  • Parameters: Optional values the method accepts.
public static int add(int a, int b) { return a + b; } // Calling a static method System.out.println(add(5, 10)); // Output: 15

Debugging

Debugging is the process of identifying and resolving issues (bugs) in the code.

Steps for Debugging:

  • Understand the Problem ‚Äî reproduce the bug consistently and understand expected vs actual behavior.
  • Use Debugging Tools ‚Äî IDEs like IntelliJ IDEA, Eclipse, and NetBeans have built-in debuggers.

Common Debugging Techniques:

  • Print Statements ‚Äî Insert System.out.println() to print variable values or track execution flow.
  • Breakpoints ‚Äî Pause execution at a specific line in an IDE to inspect the program state.
  • Step Through Code ‚Äî After setting breakpoints, step line by line to observe behavior.

Exception Handling

An exception is an event that disrupts the normal flow of a program. In Java, exceptions are objects that represent errors.

Java uses try, catch, and finally blocks:

  • try block ‚Äî contains code that might throw an exception.
  • catch block ‚Äî handles the exception. Multiple catch blocks can handle different exception types.
  • finally block ‚Äî always executes, regardless of whether an exception occurred. Used to release resources.
try { int result = 10 / 0; // may throw exception } catch (ArithmeticException e) { System.out.println("Error: " + e.getMessage()); } finally { System.out.println("This always runs."); }

throw and throws Keywords

  • throw ‚Äî used to manually throw an exception.
  • throws ‚Äî used in method signatures to declare what exceptions a method might throw.
// throw example if (age < 0) { throw new IllegalArgumentException("Age cannot be negative"); } // throws example public void readFile() throws IOException { // file reading code }

Types of Exceptions

TypeChecked AtExamples
Checked ExceptionsCompile time ‚Äî must handle with try-catch or throwsIOException, SQLException, FileNotFoundException
Unchecked Exceptions (Runtime)Runtime ‚Äî result from programming errorsNullPointerException, ArrayIndexOutOfBoundsException, ArithmeticException

Best Practices for Exception Handling

  • Catch specific exceptions ‚Äî avoid using generic Exception class.
  • Use meaningful error messages when throwing exceptions.
  • Avoid overusing exceptions ‚Äî use them only for exceptional conditions, not for control flow.
  • Release resources in a finally block or use try-with-resources.
  • Use logging frameworks (like Log4j) instead of printing stack traces.

File Handling & I/O Operations

Introduction to File Handling

File handling in Java allows programs to create, read, write, and manipulate files stored on the file system.

  • Java provides the java.io and java.nio packages for file handling.
  • Files are used to store data persistently ‚Äî configuration files, logs, user data, etc.

Key Classes for File Handling

ClassPackagePurpose
Filejava.ioManages file/directory names; check existence, create, delete. Does NOT read/write data.
FileReaderjava.ioReads character data from files
BufferedReaderjava.ioReads characters efficiently via buffering; reads line by line
FileWriterjava.ioWrites character data to files
BufferedWriterjava.ioWrites characters efficiently via buffering
PrintWriterjava.ioWrites formatted text to files (like System.out for files)
Scannerjava.utilReads and parses data from files

Basic File Operations

Creating a File ‚Äî use File class and createNewFile() method. Returns true if created, false if already exists.

Writing to a File ‚Äî use FileWriter or BufferedWriter:

try (FileWriter writer = new FileWriter("sample.txt")) { writer.write("Hello, File!"); } catch (IOException e) { System.out.println("Error: " + e.getMessage()); }

Reading from a File ‚Äî use FileReader and BufferedReader:

try (BufferedReader reader = new BufferedReader(new FileReader("sample.txt"))) { String line; while ((line = reader.readLine()) != null) { System.out.println(line); } } catch (IOException e) { System.out.println("Error: " + e.getMessage()); }

Appending to a File ‚Äî pass true as the second parameter to FileWriter:

FileWriter writer = new FileWriter("sample.txt", true); // true = append mode; does NOT overwrite existing content

Deleting a File ‚Äî use the delete() method from the File class.

Common Exceptions in File Handling

ExceptionWhen It Occurs
FileNotFoundExceptionFile does not exist or is inaccessible. Subclass of IOException.
IOExceptionGeneral failure during I/O operations (reading from closed stream, write failure, etc.)
SecurityExceptionSecurity manager denies access (no read/write permission)

Common Scenarios:

  • Missing file ‚Üí catch FileNotFoundException
  • Permission issues ‚Üí catch SecurityException or IOException
  • I/O interruption during operations ‚Üí catch IOException

Best Practices for File Handling

  • Use specific exceptions ‚Äî catch FileNotFoundException or IOException instead of a general Exception.
  • Use try-with-resources ‚Äî automatically closes resources after the try block; prevents resource leaks.
  • Provide meaningful messages ‚Äî include informative error messages or logging when catching exceptions.
  • Always close resources ‚Äî close file readers/writers to avoid memory leaks or file locks.
  • Use logging ‚Äî for production systems, log exceptions instead of printing stack traces.
  • Graceful error handling ‚Äî avoid abrupt program termination; provide fallback mechanisms (e.g., ask user for another file path).
Try-with-Resources (Best Practice) When you open a resource (like a FileWriter) inside try(...), Java automatically closes it when the block ends ‚Äî no need for a finally block.
